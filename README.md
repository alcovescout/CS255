# Briefly summarize the DriverPass project. Who was the client? What type of system did they want you to design?
DriverPass aims to provide a web portal for customers to study, test, and schedule driving lessons, in order to train for the Driver's exam. will
provide users with the ability to purchase packages, which include lessons and exams, as well as purchases in person driving lessons.

# How did you interpret the user’s needs and implement them into your system design? Why is it so important to consider the user’s needs when designing?
Ultimately DriverPass is built to serve the consumer/user, so I do think it is important to design based off of their needs. Creating use case diagrams
help better understand what the user is going to need and how we can implement it into the system. Also how those implementations will interact with the
system.
